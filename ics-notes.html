Java Object Oriented Programming (OOP)

Object oriented programming, or OOP for short, is a way of organizing a program's components into similar structures to have a customizable blueprint that suits most of each individual instance's needs.


For example, if I was to model a classroom of students, I wouldn't make a custom variable for each student and each attribute that student had in the room; instead, there would likely be a Student class to handle the structure for me, and add in customizable attributes in different class constructors or get/set methods for more flexibility depending on the instance of the class.


public class Student {
private int age;
private String name;
private double average;

public Student() {
age = 14;
name = "Dave";
average = 80;
}

public Student(int a, String n, double av) {
age = a;
name = n;
average = av;
}

public String getStudentName() {
return name;
}

public int getStudentAge() {
return name;
}

public double getStudentAverage() {
return double;
}
}

Classes

Classes are the blueprint of blueprints in Java: they provide a structure which your own custom design can follow. Classes are a type of Object in Java, which means they can be used in existing methods such as .equals, or incorporated in arrays and ArrayLists (see more on ArrayLists here). These such integrations can help with keeping track of and comparing different instances of your classes.

The header for creating a new class goes as follows: a public or private scope keyword depending on whether you want to be able to access it from elsewhere (usually using public for access from a driver class); the keyword class, followed by the name of your class. That would look something like this:

        public class Computer { // public class, accessible from other classes
        private class Computer { // private class, accessible from parent classes

Arrays

Arrays are a more compact way to handle similar groups of data of the same type. Instead of having 5 separate variables to hold 5 numbers, you can instead have an array of length 5, where each element in the array corresponds to one of these numbers. 

        averages = [80, 75, 90, 94, 83]

In Java, there are several types of arrays: primitive arrays, which hold Java's built-in data types like int and char, and Object arrays, which can be filled with instances of built-in or user-created classes, such as an array of Students, or even an array of arrays!

        grid = [[1, 3, 2, 2, 5], [3, 5, 4, 4, 3], [2, 1, 5, 1, 2]]

Initialization

There are several ways to create an array. For this demonstration, I will be using an integer array to keep things simple. The most basic way to do so is by using the following syntax: int[] array = new int[len]; This will initialize a new array of integers of length len, each of which are set to 0 by default. So if I wanted to make an array named arr with length 23, I would write:

        int[] arr = new int[23];
        arr = [0, 0, 0, ..., 0]; // initialized with all 0s

To access elements of the array, we use square brackets to surround the index of the object which we are requesting, 0-indexed, meaning the first object is in the 0th place, the second object in the 1st, and so on.

        System.out.println(arr[0]); // prints the 1st item in the list
        System.out.println(arr[3]); // prints the 4th item in the list
        System.out.println(arr[15]); // prints the 16th item in the list

Initializing an Array with Values

The most basic way to fill the array is to set each item individually:

int[] averages = new int[5];
averages[0] = 80;
averages[1] = 75;
averages[2] = 90;
averages[3] = 94;
averages[4] = 83;

While this method works, it is extremely tedious and becomes very inefficient for arrays of size 100, or 1000 and beyond. This leads to another way to fill the array upon initialization: using an array literal, which looks like this:

int[] newArray = new int[] {80, 75, 90, 94, 83};
int[] newInitializedArray = {80, 75, 90, 94, 83}; // only works for uninitialized arrays

Parallel Arrays

Parallel arrays are a way to organize information by maintaining arrays which hold information for a certain data point in the same index.

ArrayLists

While arrays in Java are constrained to a set size, ArrayLists are able to add and remove elements, but only work with Objects. They can also search for objects and check whether a given item exists.

int[] newArray = new int[] {80, 75, 90, 94, 83};
int[] newInitializedArray = {80, 75, 90, 94, 83}; // only works for uninitialized arrays
