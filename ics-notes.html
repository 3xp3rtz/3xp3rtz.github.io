Java Object Oriented Programming (OOP)

Object oriented programming, or OOP for short, is a way of organizing a program's components into similar structures to have a customizable blueprint that suits most of each individual instance's needs.


For example, if I was to model a classroom of students, I wouldn't make a custom variable for each student and each attribute that student had in the room; instead, there would likely be a Student class to handle the structure for me, and add in customizable attributes in different class constructors or get/set methods for more flexibility depending on the instance of the class.


public class Student {
private int age;
private String name;
private double average;

public Student() {
age = 14;
name = "Dave";
average = 80;
}

public Student(int a, String n, double av) {
age = a;
name = n;
average = av;
}

public String getStudentName() {
return name;
}

public int getStudentAge() {
return name;
}

public double getStudentAverage() {
return double;
}
}

Classes

Classes are the blueprint of blueprints in Java: they provide a structure which your own custom design can follow. Classes are a type of Object in Java, which means they can be used in existing methods such as .equals, or incorporated in arrays and ArrayLists (see more on ArrayLists here). These such integrations can help with keeping track of and comparing different instances of your classes.

The header for creating a new class goes as follows: a public or private scope keyword depending on whether you want to be able to access it from elsewhere (usually using public for access from a driver class); the keyword class, followed by the name of your class. That would look something like this:

        public class Computer { // public class, accessible from other classes
        private class Computer { // private class, accessible from parent classes

Arrays

Arrays are a more compact way to handle similar groups of data of the same type. Instead of having 5 separate variables to hold 5 numbers, you can instead have an array of length 5, where each element in the array corresponds to one of these numbers. 

        averages = [80, 75, 90, 94, 83]

In Java, there are several types of arrays: primitive arrays, which hold Java's built-in data types like int and char, and Object arrays, which can be filled with instances of built-in or user-created classes, such as an array of Students, or even an array of arrays!

        grid = [[1, 3, 2, 2, 5], [3, 5, 4, 4, 3], [2, 1, 5, 1, 2]]

Initialization

There are several ways to create an array. For this demonstration, I will be using an integer array to keep things simple. The most basic way to do so is by using the following syntax: int[] array = new int[len]; This will initialize a new array of integers of length len, each of which are set to 0 by default. So if I wanted to make an array named arr with length 23, I would write:

        int[] arr = new int[23];
        arr = [0, 0, 0, ..., 0]; // initialized with all 0s

To access elements of the array, we use square brackets to surround the index of the object which we are requesting, 0-indexed, meaning the first object is in the 0th place, the second object in the 1st, and so on.

        System.out.println(arr[0]); // prints the 1st item in the list
        System.out.println(arr[3]); // prints the 4th item in the list
        System.out.println(arr[15]); // prints the 16th item in the list

Initializing an Array with Values

The most basic way to fill the array is to set each item individually:

int[] averages = new int[5];
averages[0] = 80;
averages[1] = 75;
averages[2] = 90;
averages[3] = 94;
averages[4] = 83;

While this method works, it is extremely tedious and becomes very inefficient for arrays of size 100, or 1000 and beyond. This leads to another way to fill the array upon initialization: using an array literal, which looks like this:

int[] newArray = new int[] {80, 75, 90, 94, 83};
int[] newInitializedArray = {80, 75, 90, 94, 83}; // only works for uninitialized arrays

ArrayLists

While arrays in Java are constrained to a set size, ArrayLists are able to add and remove elements, but only work with Objects. They can also search for objects and check whether a given item exists.

ArrayList<Integer> intArr = new ArrayList<Integer>();
ArrayList<String> stringArr = new ArrayList<String>();
ArrayList<Object> objArr = new ArrayList<Object>();

Since ArrayLists are themselves objects, you can even do ArrayLists of ArrayLists:

        ArrayList<ArrayList<String>> intArr = new ArrayList<ArrayList<String>>();

ArrayLists are useful for lists without a set size, or data structures such as a stack, which operations are to add and remove elements. This can be replicated using the ArrayList methods add and pop as follows:

ArrayList<Integer> stack = new ArrayList<Integer>();
stack.add(10);
stack.add(5);
stack.add(32); // adding values onto the stack
System.out.println(stack.pop()); // removing values from the stack

Sorting and Searching

There are many sorting algorithms for ordering items in a data structure. The most common methods include bubble sort, insertion sort, and selection sort. These methods, while not particularly fast, are intuitive and make sense on first glance.

Bubble Sort

Bubble sort works by repeatedly going through the array and swapping any two adjacent elements which are in the wrong order.

public void bubbleSort(int[] arr) {
for (int i = 0; i < arr.length; i++) {
for (int j = i; j < arr.length-1; j++) {
if (arr[j] > arr[j+1]) { // if two elements are in the wrong order
int temp = arr[j+1];
arr[j+1] = arr[j];
arr[j] = temp; // swap them
}
}
}
}

Insertion Sort

Insertion sort works by going through the array and slowly placing each element into a sorted subarray until all elements are sorted.

public void insertionSort(int[] arr) {
for (int i = 1; i < arr.length-1; i++) {
int index = i-1;
int value = arr[i];
while (index > -1 && arr[index-1] > value) { // while value is smaller than next
arr[index+1] = arr[index];
index--;
}
arr[index+1] = value;
}
}

Source: Javatpoint Tutorial: Insertion Sort

Selection Sort

Selection sort works by going through the array finding the largest element, and adding it .

Take the following snippet as an example:

public void selectionSort(int[] arr) {
for (int i = 0; i < arr.length-1; i++) {
int index = i;
for (int j = i+1; j < arr.length; j++)
if (arr[j] < arr[index])
index = j;
int temp = arr[i];
arr[i] = arr[index];
arr[index] = temp;
}
}

Abstract Classes

An abstract class is a superclass that represents an abstract concept which should not be instantiated. However, an abstract class can have constructors which can be called from constructors of subclasses. A class with no abstract methods is called concrete.

An abstract class can be thought of as a blueprint which is always modified but never used itself. Child classes must have their own implementation of all parent abstract methods.

If a subclass of an abstract class does not provide an implementation code of ALL abstract methods it must also be declared abstract.


Take the following snippet as an example:

public class Student {
String name;
double mark;
public double getMark() { return mark; }
public double getFreeTime() {}
}

public class ElementaryStudent extends Student {
...
public double getFreeTime() { ... }
}

public class HighSchoolStudent extends Student {
...
public double getFreeTime() { ... }
}

public class UniversityStudent extends Student {
...
public double getFreeTime() { ... }
}

The Student class is the parent class, then has children ElementaryStudent, HighSchoolStudent, and UniversityStudent. However, this model is only efficient if the Student class is never instantiated.

In case we expect not to use the original Student class and specifiy which child class any given Student object should be, this is where we can use an abstract class:

public abstract class Student {
String name;
double mark;
public double getMark() { return mark; }
public abstract double getFreeTime();
}

You may have also observed that the method getFreeTime(), which was originally overrided by the child classes and was defined as a blank function, now has the keyword abstract. This simply means that the implemnentation will be shown in the child (non-abstract) classes, but polymorphism is still applicable.


Interfaces

An interface is similar to an abstract class, but has no fields or constructors and has ALL abstract methods. It is not necessary to write public abstract because all of the class's methods must be public abstract.

Interfaces are like abstract classes that don't pass down any variables, and instead can pass down public static final constants. One must "officially" state that a class implements an interface, though child classes automatically inherit interfaces from their parent classes.

public interface Edible {
String getFoodGroup();
int getCaloriesPerServing();
}

public class Pancake implements Edible {
...
}

public class Breakfast {
private int myTotalCalories = 0;
...
public void eat (Edible obj, int servings) {
myTotalCalories += obj.getCaloriesPerServing() * servings;
}
...
}
