Java Object Oriented Programming (OOP)

Object oriented programming, or OOP for short, is a way of organizing a program's components into similar structures to have a customizable blueprint that suits most of each individual instance's needs.


For example, if I was to model a classroom of students, I wouldn't make a custom variable for each student and each attribute that student had in the room; instead, there would likely be a Student class to handle the structure for me, and add in customizable attributes in different class constructors or get/set methods for more flexibility depending on the instance of the class.


public class Student {
private int age;
private String name;
private double average;

public Student() {
age = 14;
name = "Dave";
average = 80;
}

public Student(int a, String n, double av) {
age = a;
name = n;
average = av;
}

public String getStudentName() {
return name;
}

public int getStudentAge() {
return name;
}

public double getStudentAverage() {
return double;
}
}
An example of a class for a student

Scope

Scope describes the access a given method, variable, class, or other datatype has from other contexts. These are useful for limiting what external programs can access and modify information, such as trying to keep information secret.

Classes

Classes are the blueprint of blueprints in Java: they provide a structure which your own custom design can follow. Classes are a type of Object in Java, which means they can be used in existing methods such as .equals, or incorporated in arrays and ArrayLists (see more on ArrayLists here). These such integrations can help with keeping track of and comparing different instances of your classes.

The header for creating a new class goes as follows: a public or private scope keyword depending on whether you want to be able to access it from elsewhere (usually using public for access from a driver class); the keyword class, followed by the name of your class. That would look something like this:

        public class Computer { // public class, accessible from other classes
        private class Computer { // private class, accessible from parent classes

Classes also usually come with at least 1 constructor, the way to create a new instance of that object. Constructors are usually responsible for setting the initial values for the instance's attributes, which can look like the following:

public class Student {
String name;
double mark;
public Student() { // default constructor
name = "David";
mark = 90.0; // sets name and mark to some default value
}

public Student(String n) { // accepts some String input
name = n; // sets the name to the String parameter provided
mark = 90.0; // however, still sets the mark to some default value
}

public Student(String n, double m) { // accepts a String and double input
name = n;
mark = m; // sets both name and mark to the parameters provided
}

public static void main(String[] args) { // main method
Student s1 = new Student(); // first constructor: no input
Student s2 = new Student("Steve"); // second constructor: name input
Student s3 = new Student("Gary", 95.5); // third constructor: name and mark input
}
}
A short demo of different constructors



Attributes

Attributes are the pieces of information which each class holds. For example, the Student class from before had attributes age, name, and average.

Attributes in classes are declared using the private keyword, such that other classes cannot access and manipulate the variable from elsewhere, and access is provided using get and set methods. These are public methods defined within the class to return and change the value of that private variable which are available to other classes.

Without these get and set methods, the instance's variables can be manipulated from almost anywhere which can access the instance, which from a security standpoint is very unsafe, but more importantly difficult to trace.

public class Student {
private String name;
private double mark;

/** other methods and constructors not shown */

public double getMark() {
return mark;
}

public void setMark(double newMark) {
mark = newMark;
}
}
An example of get/set methods for the mark variable



Methods

Methods are certain behaviour or actions the instance of the class can perform. The examples above demonstrate the get and set methods, but methods can be much more specific to the class. For example, a Pet class could have a few methods implementated as follows:

public class Pet {
private String name;
private double food;
private double maxFood;

/** other methods and constructors not shown*/

public String getName() { return name; }
public double getFood() { return food; }
public double getMaxFood() { return maxFood; } // simple get methods

public void feed(double foodAmount) { // feeds the pet a certain amount of food
food += foodAmount;
if (food > maxFood) food = maxFood;
System.out.println(name + " was fed some food :3");
}

public boolean play() { // makes the pet play, returns whether the pet is too hungry to play
if (food < 5) {
System.out.println(name + " is too hungry to play...");
return false;
}
food -= 5;
System.out.println(name + " is playing around! ^-^");
return true;
}
}
A short demonstration on more complex behaviour by a method

All methods need to have some return type; whether it's an int, String, double, an Object like an array of integers, int[], or even no return type, in which case we use void. Without a return type, there will likely be an error where the compiler read your method as a constructor for a class with your method's name, or outright say that your method needs a return type.





Inheritance

Inheritance is specialization into a more specific structure from a larger, more general blueprint. For example, let's say you have a Pet class. This class handles all kinds of pets just fine, but what about if you want to create a lot of Pets as dogs? Instead of rewriting all of the code from the Pet class in a new Dog class, or specifying a definition of a Dog repeatedly, we can use inheritance.

Inheritance is when one class inherits qualities from another class, a relationship usually signified between a "parent" and a "child" class, where the child class inherits qualities from a parent class. One such example can be found below:

public class Pet { // the parent class
private String name;
private int age;

public Pet(String n, int a) {
name = n;
age = a;
}

public String getName() { return "Pet " + name; }
public int getAge() { return age; }
public void eat(double food) { ... }
}


public class Dog extends Pet { // the child class
private int loudness;

public Dog(String name, int age, int loud) {
super(name, age);
loudness = loud;
}

public String getName() { return name + " the Dog" }
public void bark() { ... }

public static void main(String[] args) {
Dog d = new Dog("Henry", 4, 3);
System.out.println(d.getAge()); // inherited parent method
System.out.println(d.bark()); // child method
System.out.println(d.getName()); // overridden method
}
}
A sample implementation of a Dog class as a child class of the Pet class

The most obvious difference between the Dog class and any other class is the existence of extends Pet. This is the syntax to extend a class, or to inherit from it methods, constructors, etc.

Despite inheriting the class, the Dog class is unable to access the private variables and methods in Pet as they are not in an accessible scope. However, it can still access protected, public, and variables/methods without an explicit scope keyword from its parent class.

For example, with the call to d.getAge(), there is no getAge() implemented in the Dog class, yet the code segment does not error. In fact, it calls the getAge() method from its parent class, Pet.

Abstract Classes

An abstract class is a superclass that represents an abstract concept which should not be instantiated. However, an abstract class can have constructors which can be called from constructors of subclasses. A class with no abstract methods is called concrete.

An abstract class can be thought of as a blueprint which is always modified but never used itself. Child classes must have their own implementation of all parent abstract methods.

If a subclass of an abstract class does not provide an implementation code of ALL abstract methods it must also be declared abstract.


Take the following snippet as an example:

public class Student {
String name;
double mark;
public double getMark() { return mark; }
public double getFreeTime() {}
}

public class ElementaryStudent extends Student {
...
public double getFreeTime() { ... }
}

public class HighSchoolStudent extends Student {
...
public double getFreeTime() { ... }
}

public class UniversityStudent extends Student {
...
public double getFreeTime() { ... }
}

The Student class is the parent class, then has children ElementaryStudent, HighSchoolStudent, and UniversityStudent. However, this model is only efficient if the Student class is never instantiated.

In case we expect not to use the original Student class and specifiy which child class any given Student object should be, this is where we can use an abstract class:

public abstract class Student {
String name;
double mark;
public double getMark() { return mark; }
public abstract double getFreeTime();
}

You may have also observed that the method getFreeTime(), which was originally overrided by the child classes and was defined as a blank function, now has the keyword abstract. This simply means that the implemnentation will be shown in the child (non-abstract) classes, but polymorphism is still applicable.


Interfaces

An interface is similar to an abstract class, but has no fields or constructors and has ALL abstract methods. It is not necessary to write public abstract because all of the class's methods must be public abstract.

Interfaces are like abstract classes that don't pass down any variables, and instead can pass down public static final constants. One must "officially" state that a class implements an interface, though child classes automatically inherit interfaces from their parent classes.

public interface Edible {
String getFoodGroup();
int getCaloriesPerServing();
}

public class Pancake implements Edible {
...
}

public class Breakfast {
private int myTotalCalories = 0;
...
public void eat (Edible obj, int servings) {
myTotalCalories += obj.getCaloriesPerServing() * servings;
}
...
}

Arrays

Arrays are a more compact way to handle similar groups of data of the same type. Instead of having 5 separate variables to hold 5 numbers, you can instead have an array of length 5, where each element in the array corresponds to one of these numbers. 

        averages = [80, 75, 90, 94, 83]

In Java, there are several types of arrays: primitive arrays, which hold Java's built-in data types like int and char, and Object arrays, which can be filled with instances of built-in or user-created classes, such as an array of Students, or even an array of arrays!

        grid = [[1, 3, 2, 2, 5], [3, 5, 4, 4, 3], [2, 1, 5, 1, 2]]

Initialization

There are several ways to create an array. For this demonstration, I will be using an integer array to keep things simple. The most basic way to do so is by using the following syntax: int[] array = new int[len]; This will initialize a new array of integers of length len, each of which are set to 0 by default. So if I wanted to make an array named arr with length 23, I would write:

        int[] arr = new int[23];
        arr = [0, 0, 0, ..., 0]; // initialized with all 0s

To access elements of the array, we use square brackets to surround the index of the object which we are requesting, 0-indexed, meaning the first object is in the 0th place, the second object in the 1st, and so on.

        System.out.println(arr[0]); // prints the 1st item in the list
        System.out.println(arr[3]); // prints the 4th item in the list
        System.out.println(arr[15]); // prints the 16th item in the list

Initializing an Array with Values

The most basic way to fill the array is to set each item individually:

int[] averages = new int[5];
averages[0] = 80;
averages[1] = 75;
averages[2] = 90;
averages[3] = 94;
averages[4] = 83;

While this method works, it is extremely tedious and becomes very inefficient for arrays of size 100, or 1000 and beyond. This leads to another way to fill the array upon initialization: using an array literal, which looks like this:

int[] newArray = new int[] {80, 75, 90, 94, 83};
int[] newInitializedArray = {80, 75, 90, 94, 83}; // only works for uninitialized arrays


Parallel Arrays

Parallel arrays are a way to organize information by maintaining arrays which hold information for a certain data point in the same index.

int[] students = {345310214, 323910324, 369851240, 321823359};
int[] ages = {15, 14, 16, 16};
double[] averages = {93.2, 94.4, 98.6, 94.5};
for (int i = 0; i < students.length; i++) {
System.out.println("The student " + students[i] + ", age " + ages[i] + ", has an average of " + averages[i]);
}

In the example above, information is stored in separate arrays, however the information for any given student is stored in the same index of the other arrays as the index of the student within the students array.


Recursion

Recursion is essentially a snippet of code which calls itself (often conditionally, to avoid an infinite loop). A function performs certain actions then calls that same function within itself.

public int factorial(int iters) {
if (iters > 1) return iters * factorial(iters-1);
return 1;
}

System.out.println(factorial(5)); // prints the 5 factorial, 120

The code snippet above performs the factorial function (the multiplication of the first n numbers) recursively, as you can see it calls itself within the if statement. The usage for recursion can be very useful for tasks of recursive nature, which includes the Fibonacci sequence, factorials, GCD, and more.

Recursion can also help in other cases, such as traversing a graph using methods like BFS (breadth-first search) and DFS (depth-first search).

ArrayLists

While arrays in Java are constrained to a set size, ArrayLists are able to add and remove elements, but only work with Objects. They can also search for objects and check whether a given item exists.

ArrayList<Integer> intArr = new ArrayList<Integer>();
ArrayList<String> stringArr = new ArrayList<String>();
ArrayList<Object> objArr = new ArrayList<Object>();

Since ArrayLists are themselves objects, you can even do ArrayLists of ArrayLists:

        ArrayList<ArrayList<String>> intArr = new ArrayList<ArrayList<String>>();

ArrayLists are useful for lists without a set size, or data structures such as a stack, which operations are to add and remove elements. This can be replicated using the ArrayList methods add and pop as follows:

ArrayList<Integer> stack = new ArrayList<Integer>();
stack.add(10);
stack.add(5);
stack.add(32); // adding values onto the stack
System.out.println(stack.pop()); // removing values from the stack

Sorting

There are many sorting algorithms for ordering items in a data structure. The most common methods include bubble sort, insertion sort, and selection sort. These methods, while not particularly fast, are intuitive and make sense on first glance.

Bubble Sort

Bubble sort works by repeatedly going through the array and swapping any two adjacent elements which are in the wrong order.

public void bubbleSort(int[] arr) {
for (int i = 0; i < arr.length; i++) {
for (int j = i; j < arr.length-1; j++) {
if (arr[j] > arr[j+1]) { // if two elements are in the wrong order
int temp = arr[j+1];
arr[j+1] = arr[j];
arr[j] = temp; // swap them
}
}
}
}

Source: Javatpoint Tutorial: Bubble Sort

Insertion Sort

Insertion sort works by going through the array and slowly placing each element into a sorted subarray until all elements are sorted.

public void insertionSort(int[] arr) {
for (int i = 1; i < arr.length-1; i++) {
int index = i-1;
int value = arr[i];
while (index > -1 && arr[index-1] > value) { // while value is smaller than next
arr[index+1] = arr[index];
index--;
}
arr[index+1] = value;
}
}

Source: Javatpoint Tutorial: Insertion Sort

Selection Sort

Selection sort works by going through the array finding the smallest element in an unsorted part of the array, and adding it to the front of the array, behind all already sorted elements.

Take the following snippet as an example:

public void selectionSort(int[] arr) {
for (int i = 0; i < arr.length-1; i++) {
int index = i;
for (int j = i+1; j < arr.length; j++) // finding the smallest element
if (arr[j] < arr[index])
index = j;
int temp = arr[i];
arr[i] = arr[index];
arr[index] = temp; // swap the smallest element with the first element of the unsorted array
}
}

Source: Javatpoint Tutorial: Selection Sort


Searching

Once we have a sorted data structure, we can use the sorted nature in order to search and find elements of our choosing. The most obvious example is binary search, which is a sort of "higher-lower" searching approach to find a given element within a sorted array. The algorithm could be implemented as follows:

// recursive implementation public int binarySearch(int[] arr, int target, int l, int r) {
if (r < l) return -1;
if (r == l) return arr[r] == arr[l];
for (int j = i+1; j < arr.length; j++) // finding the smallest element
if (arr[j] < arr[index])
index = j;

}
